# ISO Structure and Boot Process

## ISO Layout

When `grub-mkrescue` finishes, the resulting `tajaos.iso` contains:

```
tajaos.iso
├── boot/
│   ├── vmlinuz          ← Linux kernel (6.8.x)
│   ├── initrd.img       ← Initramfs with live-boot
│   └── grub/
│       └── grub.cfg     ← GRUB configuration
├── live/
│   └── filesystem.squashfs  ← Compressed root filesystem
└── [EFI/]               ← UEFI boot partition (grub-mkrescue)
```

## Boot Flow

1. **BIOS/UEFI** loads GRUB from the ISO's boot record.
2. **GRUB** presents a menu (default: "TajaOS v2.0", timeout 5s).
3. GRUB loads `vmlinuz` + `initrd.img` with kernel parameters: `boot=live components quiet hostname=tajaos`.
4. The **initramfs** (generated by live-boot) detects the ISO, mounts `filesystem.squashfs`, and sets up an overlay filesystem.
5. **systemd** starts and executes the default target.
6. **agetty** auto-logs in as root on tty1.
7. **Bash** starts with the configured `.bashrc` and `.bash_profile`.

## Kernel Parameters

| Param | Purpose |
|---|---|
| `boot=live` | Tells initramfs to use live-boot |
| `components` | Enables multiple rootfs components |
| `quiet` | Suppresses most boot messages |
| `hostname=tajaos` | Sets the system hostname |
| `toram` | (RAM mode) Copies the entire ISO to RAM |

## BIOS vs UEFI

`grub-mkrescue` creates an ISO that boots on both:
- **BIOS** — via El Torito boot record with `-boot-load-size 4`
- **UEFI** — via an embedded FAT partition with `grub-efi-amd64-bin`

No manual `bios.img` or `efiboot.img` is needed.
